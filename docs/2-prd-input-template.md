## 작성 가이드라인

### 답변 작성 시 유의사항
1. **구체성**: "빠르다"보다 "3초 이내"로 수치화
2. **완전성**: 모든 질문에 답변하되, 불확실한 경우 "TBD [이유]"로 표시
3. **일관성**: 섹션 간 상충되는 내용이 없도록 확인
4. **현실성**: 달성 가능한 목표와 일정 설정

### 작성 분량 가이드
- 제품 비전: 2-3문장
- 페르소나: 각 200-300자
- 사용자 여정: 5-10단계
- 기술 스택: 각 항목별 선택 이유 1-2문장

## 5. 기능 우선순위

### 5.1 MVP (Minimum Viable Product) 기능

**질문**: 도메인 정의서의 UC-01~UC-09 중 MVP에 반드시 포함되어야 하는 기능은 무엇인가요?

| 유스케이스            | 우선순위 | MVP 포함 여부 | 사유                            |
| --------------------- | -------- | ------------- | ------------------------------- |
| UC-01: 회원가입       | MUST     | ✅            | 필수 기능                       |
| UC-02: 로그인         | MUST     | ✅            | 필수 기능                       |
| UC-03: 할일 목록 조회 | MUST     | ✅            | 핵심 기능                       |
| UC-04: 할일 추가      | MUST     | ✅            | 핵심 기능                       |
| UC-05: 할일 수정      | MUST     | ✅            | 핵심 기능                       |
| UC-06: 할일 삭제      | SHOULD   |               | 복구 기능과 함께 구현 예정      |
| UC-07: 할일 복구      | SHOULD   |               | 차별화 기능이지만 우선순위 낮음 |
| UC-08: 할일 영구 삭제 | COULD    |               | Phase 2 고려                    |
| UC-09: 휴지통 조회    | COULD    |               | Phase 2 고려                    |

> 우선순위: MUST (필수), SHOULD (권장), COULD (가능하면), WON'T (제외)
