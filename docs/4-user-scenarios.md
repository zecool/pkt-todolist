## 1. 개요

### 1.1 문서 목적

이 문서는 pkt-todolist 애플리케이션의 실제 사용 시나리오를 구체적으로 기술하여:
- 개발팀이 사용자 관점에서 기능을 이해할 수 있도록 함
- QA 팀이 테스트 케이스를 작성할 수 있도록 함
- 사용자 경험(UX)을 개선하기 위한 인사이트 제공

### 1.2 시나리오 작성 형식

각 시나리오는 다음 형식으로 작성됩니다:

```
시나리오 이름: [간단한 제목]
페르소나: [사용자 페르소나]
전제 조건: [시나리오 시작 전 상태]
목표: [사용자가 달성하려는 목표]

단계:
1. [사용자 액션] → [시스템 응답]
2. ...

결과: [최종 상태 및 사용자 만족도]
관련 UC: [유스케이스 ID]
```

### 1.3 주요 페르소나

**PRD 3장 참조**:

1. **김철수** (20-30대 직장인)
   - 업무 일정 관리
   - 하루 10분-1시간 사용
   - 간단하고 명료한 인터페이스 선호

2. **카리나** (20대 초반 대학생)
   - 과제 및 시험 일정 관리
   - 아침에 주로 사용
   - 모바일 중심

#### 시나리오 2.1.1: 출근길 할일 확인 및 추가

**페르소나**: 김철수 (직장인)
**시간**: 월요일 오전 8:30, 지하철에서
**전제 조건**: 이미 가입 및 로그인한 상태
**목표**: 오늘 할 일 확인하고 새로운 업무 할일 추가

**단계**:

1. **지하철에서 스마트폰으로 앱 접속**
   - 앱 열기 → 로그인 상태 유지 (JWT 토큰)
   - 메인 화면(할일 목록) 자동 표시

2. **오늘 할일 목록 확인**
   - "월요일 회의 준비" (만료일: 오늘)
   - "주간 보고서 작성" (만료일: 11-27)
   - "클라이언트 이메일 답장" (완료됨, 체크 표시)

3. **긴급 업무 추가**
   - 화면 하단 "+" 버튼 터치
   - 제목: "팀장님께 보고서 제출"
   - 내용: "오전 11시까지 제출 필요"
   - 시작일: 오늘 (2025-11-25)
   - 만료일: 오늘 (2025-11-25)
   - "저장" 버튼 터치

4. **할일 추가 확인**
   - 목록 최상단에 새 할일 표시
   - 만료일이 오늘이므로 주황색 강조

**결과**:
- ✅ 3개의 할일 목록 (1개 완료, 2개 진행 중)
- ✅ 긴급 업무를 놓치지 않고 관리
- ✅ 소요 시간: 약 2분

**관련 UC**: [UC-03] 할일 조회, [UC-04] 할일 추가

#### 시나리오 2.1.3: 실수로 삭제한 할일 복원

   - 확인 다이얼로그: "휴지통으로 이동하시겠습니까?"

5. **할일 복원**
   - "주간 보고서 작성" 항목의 "복원" 버튼 클릭
   - 토스트 메시지: "할일이 복원되었습니다"
   - 자동으로 할일 목록으로 이동

6. **복원 확인**
   - 할일 목록에 "주간 보고서 작성" 다시 표시
   - 원래 상태 그대로 유지

**결과**:
- ✅ 실수로 삭제한 중요 할일 복구
- ✅ 데이터 손실 없이 안심
- ✅ 소요 시간: 1분

## 3. 기능별 상세 시나리오

### 3.1 회원가입 및 첫 로그인

#### 시나리오 3.1.1: 신규 사용자 회원가입

**페르소나**: 신규 사용자 (이름: 박지민)
**전제 조건**: 앱에 처음 방문
**목표**: 계정 생성하고 서비스 이용 시작

**단계**:

1. **앱 접속**
   - 브라우저에서 URL 입력 또는 검색
   - 랜딩 페이지 표시: "pkt-todolist"
   - "시작하기" 또는 "회원가입" 버튼

2. **회원가입 페이지**
   - 이메일 입력: "jimin@example.com"
   - 비밀번호 입력: "SecurePass123!"
   - 비밀번호 확인: "SecurePass123!"
   - 사용자 이름: "박지민"

3. **유효성 검증**
   - 이메일 형식 확인 ✅
   - 비밀번호 길이 확인 (최소 8자) ✅
   - 비밀번호 일치 확인 ✅

4. **회원가입 완료**
   - "회원가입" 버튼 클릭
   - 서버 요청: POST /api/auth/register
   - 성공 응답: userId, 자동 로그인

5. **첫 화면 진입**
   - "환영합니다, 박지민님!" 환영 메시지
   - 빈 할일 목록 화면

**목표**: 데스크톱과 모바일 간 끊김 없는 동기화

1. **사무실 데스크톱에서 할일 추가**
   - 시간: 오후 5:00
   - "퇴근 후 마트 가기" 할일 추가
   - 만료일: 오늘

2. **퇴근 후 지하철에서 모바일 확인**
   - 시간: 오후 6:00
   - 스마트폰으로 앱 접속
   - 자동 로그인 (토큰 유지)
   - "퇴근 후 마트 가기" 할일 표시 ✅

3. **마트 도착 후 완료 처리**
   - 시간: 오후 6:30
   - 모바일에서 체크박스 터치
   - 완료 처리

4. **집에서 데스크톱 확인**
   - 시간: 오후 7:00
   - 데스크톱 브라우저 새로고침
   - "퇴근 후 마트 가기" 완료 상태 동기화 ✅

**결과**:
- ✅ 데스크톱 ↔ 모바일 자동 동기화
