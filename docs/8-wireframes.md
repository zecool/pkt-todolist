### 1.3 주요 컴포넌트

- **Input** (이메일, 비밀번호)
  - 타입: text, password
  - 검증: 이메일 형식, 필수 입력
  - 에러 상태 표시 (빨간 테두리)
- **Button** (로그인)
  - 타입: Primary
  - 색상: 네이버 그린 (#00C73C)
  - 상태: 기본, 호버, 비활성
- **Link** (회원가입)
  - 색상: 회색 텍스트 + 언더라인

### 1.4 인터랙션

1. 이메일 입력 → 포커스 시 테두리 강조
2. 비밀번호 입력 → 마스킹 처리
3. 로그인 버튼 클릭:
   - API 요청: `POST /api/auth/login`
   - 성공 → JWT 토큰 저장 → 메인 페이지(`/`) 이동
   - 실패 → 에러 메시지 표시 (입력 필드 하단)
4. 회원가입 링크 클릭 → `/register` 페이지 이동

### 1.5 사용자 플로우

```
[로그인 페이지 진입]
     ↓
[이메일/비밀번호 입력]
     ↓
[로그인 버튼 클릭]
     ↓
[인증 처리] → 성공 → [메인 페이지]
     ↓
   실패 → [에러 메시지 표시] → [재입력]
```

### 1.6 접근성 고려사항

- 이메일 필드: `aria-label="이메일 주소"`, `type="email"`
- 비밀번호 필드: `aria-label="비밀번호"`, `type="password"`
- 에러 메시지: `role="alert"`, 스크린 리더 읽기
- 키보드 네비게이션: Tab 키로 포커스 이동, Enter 키로 로그인

### 2.3 주요 컴포넌트

- **Input** (이름, 이메일, 비밀번호, 비밀번호 확인)
  - 타입: text, email, password
  - 검증:
    - 이름: 필수 입력
    - 이메일: 형식 검증, 중복 검사
    - 비밀번호: 최소 8자
    - 비밀번호 확인: 일치 여부
  - 실시간 검증 피드백 (아이콘 + 메시지)
- **Button** (회원가입)
  - 타입: Primary
  - 색상: 네이버 그린 (#00C73C)

### 2.4 인터랙션

1. 입력 필드별 실시간 검증:
   - 이메일 형식 → ✅ 또는 ❌ 아이콘
   - 비밀번호 길이 → 8자 미만 시 경고
   - 비밀번호 일치 → 실시간 비교
2. 회원가입 버튼 클릭:
   - 모든 필드 검증 통과 확인
   - API 요청: `POST /api/auth/register`
   - 성공 → 자동 로그인 → 메인 페이지 이동 + 환영 메시지
   - 실패 → 에러 메시지 (예: "이미 사용 중인 이메일입니다")
3. 로그인 링크 클릭 → `/login` 페이지 이동

### 2.5 사용자 플로우

```

     ↓

     ↓

     ↓
   실패 → [에러 메시지 표시] → [재입력]
```

### 2.6 접근성 고려사항

- 필드별 `aria-label`, `aria-describedby` (에러 메시지 연결)
- 비밀번호 강도 표시 (선택 사항)
- 에러 메시지: `role="alert"`

### 3.3 주요 컴포넌트

#### Header (상단 바)
- 로고: "pkt-todolist"
- 검색 아이콘 (모바일: 상단, 데스크톱: 우측)
- 프로필 드롭다운: 사용자명 + 화살표

- 메뉴 항목:
  - 📋 할일
  - 🗑️ 휴지통

  - 👤 프로필
- 선택된 메뉴 강조 (네이버 그린 배경)

#### TodoCard (할일 카드)
- 체크박스: 완료 처리
- 제목: 볼드체
- 내용: 작은 글씨, 회색
- 날짜 및 상태 표시:
  - 📅 만료일
  - 상태 아이콘: 🔴 긴급 (오늘 마감), 🟠 진행중, ✅ 완료
- 액션 버튼: [수정] [삭제]

#### Button (추가)
- 위치:
  - 데스크톱: 우측 상단 "+ 추가"
  - 모바일: 하단 고정 플로팅 버튼 "➕"
- 색상: 네이버 그린 (#00C73C)
- 클릭 → 할일 추가 모달 열림

#### 필터 탭
- 탭 메뉴: [전체 | 진행중 | 완료]

### 3.4 인터랙션

1. **할일 체크박스 클릭**:
   - API 요청: `PATCH /api/todos/:id/complete`
   - 체크 애니메이션
   - 카드 색상 변경 (회색 배경, 초록 체크)
   - 완료된 항목은 하단으로 이동 (정렬)

2. **수정 버튼 클릭**:
   - 할일 상세/수정 모달 열림

3. **삭제 버튼 클릭**:
   - 확인 다이얼로그: "휴지통으로 이동하시겠습니까?"
   - 확인 → API 요청: `DELETE /api/todos/:id`
   - 카드가 페이드아웃 애니메이션과 함께 사라짐
   - 토스트 메시지: "할일이 휴지통으로 이동되었습니다"

4. **+ 추가 버튼 클릭**:
   - 할일 추가 모달 열림

5. **필터 탭 전환**:
   - [전체]: 모든 할일 표시
   - [진행중]: `status='active'` 할일만 표시
   - [완료]: `status='completed'` 할일만 표시

6. **검색**:
   - 검색어 입력 → 실시간 필터링
   - API 요청: `GET /api/todos?search=검색어`

  - 로그아웃 버튼 (데스크톱)
- **반응형**:
  - 데스크톱: 전체 메뉴 표시
  - 모바일: 햄버거 메뉴 또는 아이콘만 표시

#### Sidebar (사이드바)
- **위치**: 데스크톱 좌측
- **구성**:
  - 메뉴 항목:
    - 📋 할일
    - 🗑️ 휴지통
    - 📅 국경일
    - 👤 프로필
    - 🔧 관리자 (관리자만)
- **반응형**:
  - 데스크톱: 항상 표시
  - 모바일: 하단 네비게이션으로 대체

#### Button (버튼)
- **타입**:
  - Primary: 네이버 그린 (#00C73C), 흰색 텍스트
  - Secondary: 회색 배경, 검은 텍스트
  - Danger: 빨간색 (#E53935), 흰색 텍스트
  - Icon: 아이콘만 (예: + 버튼)
- **상태**:
  - 기본: 정상 색상
  - 호버: 밝기 증가
  - 비활성: 회색, 클릭 불가
  - 포커스: 파란 테두리 (접근성)

#### Input Field (입력 필드)
- **타입**: text, email, password, date
- **상태**:
  - 기본: 회색 테두리
  - 포커스: 네이버 그린 테두리
  - 에러: 빨간 테두리 + 하단 에러 메시지
  - 읽기 전용: 회색 배경, 텍스트만 표시

#### Modal (모달)
- **구성**:
  - 배경 오버레이: 어두운 반투명 배경
  - 모달 박스: 중앙 정렬 (데스크톱), 풀스크린 (모바일)
  - 닫기 버튼: [✕] 우측 상단
- **애니메이션**:
  - 열림: 페이드인 + 스케일 증가
  - 닫힘: 페이드아웃 + 스케일 감소

#### TodoCard (할일 카드)
- **구성**:
  - 체크박스: 왼쪽
  - 제목: 볼드체
  - 내용: 작은 글씨, 회색
  - 날짜 및 상태: 📅 아이콘 + 텍스트
  - 액션 버튼: [수정] [삭제] (우측)
- **색상**:
  - 진행 중: 주황색 왼쪽 바 (#FF7043)
  - 완료: 초록색 배경 (#66BB6A), 체크 표시
  - 삭제: 회색 (#BDBDBD)
  - 긴급 (오늘 마감): 빨간색 강조 (#E53935)

#### Toast (토스트 메시지)
- **위치**: 화면 상단 중앙 또는 하단 (모바일)
- **타입**:
  - 성공: 초록색 배경 + 체크 아이콘
  - 에러: 빨간색 배경 + 경고 아이콘
  - 정보: 파란색 배경 + 정보 아이콘
- **애니메이션**: 슬라이드인 → 3초 후 슬라이드아웃

## 13. 애니메이션 및 전환

### 13.1 페이지 전환
- 페이드인: 200ms
- 슬라이드인: 300ms (모바일 페이지)

### 13.2 할일 카드 애니메이션
- 체크 시: 체크 애니메이션 + 배경 색상 전환 (300ms)
- 삭제 시: 페이드아웃 + 슬라이드 (400ms)
- 추가 시: 슬라이드 다운 (300ms)

### 13.3 모달 애니메이션
- 열림: 페이드인 + 스케일 (0.95 → 1.0, 200ms)
- 닫힘: 페이드아웃 + 스케일 (1.0 → 0.95, 200ms)
